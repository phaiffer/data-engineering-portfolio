# Marts Reference

All marts are materialized as BigQuery tables in the mart dataset generated by dbt, usually:

```text
<project>.stackoverflow_analytics_marts
```

The marts are analytical outputs for BigQuery inspection and downstream BI readiness. They are not
API payloads and not dashboard-specific tables.

## Reviewer Lens

When reviewing a mart, check:

- **Grain:** what one row represents.
- **Question answered:** why the mart exists.
- **Cost behavior:** whether downstream users can query a persisted table instead of re-scanning sources.
- **Caveat:** what the mart intentionally does not claim.

## Mart Summary

| Mart | Grain | Question answered | Main caveat |
|---|---|---|---|
| `mart_monthly_question_activity` | Calendar month | How are question volume, answer rates, and response times changing? | Windowed by `stackoverflow_min_year` |
| `mart_tag_activity` | Tag | Which technology ecosystems are most active and healthy? | Uses a 10-question threshold |
| `mart_answer_latency` | Calendar month | How quickly does the community respond? | Percentiles are approximate |
| `mart_question_outcomes` | Question | What happened to each question in the selected window? | Excludes raw HTML body text |
| `mart_user_reputation_segments` | Reputation tier | How does user experience relate to question outcomes? | Only users asking questions in the window |

## mart_monthly_question_activity

**Grain:** One row per calendar month within the staging window.

**Primary use:** Track question volume, engagement, answer rates, and resolution trends over time.

**Cost behavior:** Persisted as a table so trend queries do not re-scan the source posts tables.

Key columns:

| Column | Description |
|---|---|
| `month_start` | First day of the calendar month |
| `total_questions` | Questions created in the month |
| `unanswered_questions` | Questions with zero answers |
| `questions_with_accepted_answer` | Questions with an accepted answer |
| `unanswered_rate_pct` | Percent of questions unanswered |
| `accepted_answer_rate_pct` | Percent of questions with an accepted answer |
| `avg_hours_to_first_answer` | Average hours to first answer |
| `median_hours_to_first_answer` | Approximate median hours to first answer |

Example review query:

```sql
select
  month_start,
  total_questions,
  unanswered_rate_pct,
  accepted_answer_rate_pct,
  median_hours_to_first_answer
from `your-project-id.stackoverflow_analytics_marts.mart_monthly_question_activity`
order by month_start desc
limit 12;
```

**Important caveat:** This mart uses the configured staging window. It is not a full historical
trend unless `stackoverflow_min_year` is set to the first available source year.

## mart_tag_activity

**Grain:** One row per tag with at least 10 questions in the selected window.

**Primary use:** Identify active technology ecosystems and compare resolution or unanswered rates.

**Cost behavior:** Tag explosion is isolated upstream and the final tag metrics are persisted.

Key columns:

| Column | Description |
|---|---|
| `tag_name` | Stack Overflow tag slug |
| `questions_in_window` | Questions with this tag in the selected window |
| `questions_with_accepted_answer` | Windowed questions resolved with accepted answers |
| `unanswered_questions` | Windowed questions with no answers |
| `accepted_answer_rate_pct` | Percent of tag questions with an accepted answer |
| `unanswered_rate_pct` | Percent of tag questions unanswered |
| `total_historical_question_count` | All-time source tag count |

Example review query:

```sql
select
  tag_name,
  questions_in_window,
  unanswered_rate_pct,
  accepted_answer_rate_pct,
  total_historical_question_count
from `your-project-id.stackoverflow_analytics_marts.mart_tag_activity`
order by questions_in_window desc
limit 20;
```

**Important caveat:** `questions_in_window` is scoped to the configured year window, while
`total_historical_question_count` is all-time from the source tags table.

## mart_answer_latency

**Grain:** One row per calendar month for questions with at least one answer.

**Primary use:** Analyze response-time distributions and track how quickly the community responds.

**Cost behavior:** Uses BigQuery `APPROX_QUANTILES` for cheaper percentile metrics.

Key columns:

| Column | Description |
|---|---|
| `month_start` | First day of the calendar month |
| `questions_with_answers` | Answered questions in this month |
| `questions_with_accepted_answer` | Questions with an accepted answer |
| `avg_hours_to_first_answer` | Average hours to first answer |
| `p25_hours_to_first_answer` | Approximate 25th percentile latency |
| `median_hours_to_first_answer` | Approximate median latency |
| `p75_hours_to_first_answer` | Approximate 75th percentile latency |
| `p90_hours_to_first_answer` | Approximate 90th percentile latency |

Example review query:

```sql
select
  month_start,
  questions_with_answers,
  p25_hours_to_first_answer,
  median_hours_to_first_answer,
  p75_hours_to_first_answer,
  p90_hours_to_first_answer
from `your-project-id.stackoverflow_analytics_marts.mart_answer_latency`
order by month_start desc
limit 12;
```

**Important caveat:** Percentiles are approximate by design. The goal is trend-level warehouse
analytics, not exact latency audit reporting.

## mart_question_outcomes

**Grain:** One row per question in the staging window.

**Primary use:** Question-level exploration, segmentation, and validation of outcome categories.

**Cost behavior:** Provides a persisted question fact table without carrying large text columns.

Key columns:

| Column | Description |
|---|---|
| `question_id` | Unique question post ID |
| `question_title` | Question title |
| `tags_clean` | Pipe-delimited tag string |
| `question_score` | Net vote score |
| `view_count` | Cumulative view count |
| `answer_count` | Number of answers at source refresh time |
| `has_accepted_answer` | Whether an accepted answer exists |
| `is_unanswered` | Whether `answer_count` is zero |
| `hours_to_first_answer` | Hours to first answer, null when unanswered |
| `hours_to_accepted_answer` | Hours to accepted answer, null when not accepted |
| `outcome_category` | `resolved`, `answered_no_accept`, `unanswered`, or `unknown` |
| `score_tier` | `viral`, `popular`, `good`, `decent`, `neutral`, or `downvoted` |
| `created_month_start` | First day of the creation month |

Example review query:

```sql
select
  outcome_category,
  score_tier,
  count(*) as question_count
from `your-project-id.stackoverflow_analytics_marts.mart_question_outcomes`
group by 1, 2
order by question_count desc;
```

**Important caveat:** This table excludes the raw HTML question body and should not be used for NLP
or text mining without a separate text-specific modeling pass.

## mart_user_reputation_segments

**Grain:** One row per reputation tier.

**Primary use:** Understand how asker experience relates to question quality and resolution.

**Cost behavior:** Aggregates question behavior by a small fixed tier dimension.

Key columns:

| Column | Description |
|---|---|
| `reputation_tier` | `new`, `contributor`, `established`, `advanced`, or `expert` |
| `tier_rank` | Display order from 1 to 5 |
| `user_count` | Users in the tier who asked questions in the window |
| `total_questions` | Total questions from users in this tier |
| `questions_with_accepted_answer` | Questions resolved with an accepted answer |
| `unanswered_questions` | Unanswered questions from this tier |
| `avg_questions_per_user` | Average questions per user in this tier |
| `avg_question_score` | Average question score |
| `avg_question_views` | Average view count |
| `avg_accepted_answer_rate_pct` | Average accepted-answer rate |

Tier thresholds:

| Tier | Reputation range |
|---|---|
| `new` | < 100 |
| `contributor` | 100-999 |
| `established` | 1,000-9,999 |
| `advanced` | 10,000-99,999 |
| `expert` | >= 100,000 |

Example review query:

```sql
select
  reputation_tier,
  tier_rank,
  user_count,
  total_questions,
  avg_accepted_answer_rate_pct,
  avg_question_score
from `your-project-id.stackoverflow_analytics_marts.mart_user_reputation_segments`
order by tier_rank;
```

**Important caveat:** This mart summarizes users who asked questions in the configured window. It is
not a complete segmentation of all Stack Overflow users.
